Is finding as easy as checking?

P vs NP asks whether every problem whose solution can be quickly verified can also be quickly found. Explore the core ideas with intuition-first explanations and interactive sandboxes.

NP-Complete

The "hardest" problems in NP — a fast solution to one solves them all.

Why it matters

  • Real-world impact

    From logistics and chip design to biology and cryptography, many tasks reduce to NP-complete problems.

  • All or nothing

    If any NP-complete problem has a fast algorithm, then every problem in NP does too.

  • Security implications

    Modern cryptography relies on some problems being hard to solve quickly.

Core intuitions

Search vs. Verify

Verifying a proposed solution is often much easier than finding one. Think of checking a Sudoku vs. solving it.

Reductions

To show a problem is hard, reduce a known hard problem to it: a general recipe for transferring difficulty.

Completeness

NP-complete problems are the hubs of difficulty: crack one, and the rest follow by reductions.

Complement graph intuition

            
If G has a clique of size k, then the complement graph Ḡ has an independent set of size k. The complement of an independent set is a vertex cover of size |V|−k in G.

Encoding decisions as digits

            
Classic reduction encodes truth assignments into digits across columns so that selecting certain numbers satisfies all clauses if and only if the sum matches a constructed target.
